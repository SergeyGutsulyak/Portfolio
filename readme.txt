Proj1 - Игра на JS (ES5) отрисовка в canvas...управление стрелками вверх, вправо, влево и свайпами если с телефона.
        По нажатии на клавишу вверх, движется с ускорением...тормозится по отпускании кнопки вверх "встроенным" притяжением.
	При столкновении с препятствием на скорости выше заданно тарелка терпит крушение.При движении расходуется топливо. 
	Загрузка уровня по AJAX в формате JSON с сервера it-academy. На проект было меньше двух недель по вечерам, если будет реальным
	препятствием, доукрашу и выведу скорость, топливо и более четко обозначу столкновения.

Proj2 back- В базу SQLLite (перенастроить можно было и на другую) через SQLAlchemy записываются ID групп соцсети VK. Приложение через
	API VK получает участников указанных групп и их ID записывает в базу. При следующем сканировании участники снова считываются 
	из VK и сравниваются с теми, что уже есть в базе. Новых и удаленных записывает в отдельную таблицу, а текущее состояние сохроняет.
	В итоге получается исотрия по участникам для выбранных групп. Нужна для попытка анализа рекламы, поиска аудитории у конкурентов и так
	может клиенту не понравилось что, и он ушел. Все больше для эксперимента и показывать не планировалось. Для сканирования групп нужно прописать
	свой токен. Запуск vkproj3.py Делалось с использованием Flask.Python. 

Proj3 front -Визуализация сканируемых групп и участников из proj2 back. React+Redux+Route. Обмен данными в формате JSON. У проекта было около 10 критериев
	оценки, среди которых не было слова красиво, со всеми вытекающмими.
